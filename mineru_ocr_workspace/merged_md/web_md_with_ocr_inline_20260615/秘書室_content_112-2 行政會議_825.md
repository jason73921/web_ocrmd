## 元智大學校務自主改善管理辦法（草案）

112.08.23 112 學年度第 2 次行政會議審議

第一條 為鼓勵全校教職員生對於校務推動，透過由下而上自主改善方式，提昇服務品質與服務滿意度，增進校務運作效率與效能，特訂定本自主改善管理辦法（以下簡稱本辦法）。

第二條 全校教職員生皆可提出校務自主改善案，並自行組織團隊進行改善提案規劃、改善工作推動與改善成果分析。

第三條 校務自主改善內容包括以下之任一項目：一、改善管理制度、行政流程、法規章程；二、提升服務品質、服務滿意度；三、提高工作品質、工作士氣；四、其他足以提升校務營運績效事項。

第四條 校務研究中心為校務自主改善提案之受理單位，考量自主改善之有效性與及時處理，採隨到隨審方式，進行改善提案暨改善成果之審查。

第五條 為求自主改善案審查與推動過程之周全，校務研究中心得邀請業務主辦單位，或相關專長之校內專兼任教師與職技人員，參與自主改善案之審查或推動。

第六條 自主改善提案經審查通過發給獎勵金貳仟元，並核撥改善過程所需之基本預算；改善成果獎勵金則依據改善成果進行審查，並提交校務研究指導委員會核備後發給。

第七條 改善工作推動暨改善成果分析期程以三個月為原則，最長不得超過六個月。

第八條 本辦法補助自主改善案數量以該學年度通過之預算額度為限。

第九條 本辦法經行政會議通過後公布實施，修正時亦同。

## 三、教育推廣

1．課程及宣導活動外，資訊及網路智財權推廣具體作法概述如下：

(1）凡未經授權使用之非法軟體，不得安裝於公用電腦及行政電腦上，經發現後將要求移除，或者請使用者購買合法授權。校內教職員生可於「校園授權軟體下載」網頁中下載微軟系列的Office、SQLServer、Windows、WindowsServer，以及統計軟體SAS。

(2）針對學生自行安裝未經授權軟體，而違反資訊安全規範者，採取網路停權、勸導、宣傳，並請學生自行移除不合法軟體。

(3)辦理有關智財權相關座談會、法律宣導活動。

(4)設置法律專家顧問，聘請李怡卿律師長期擔任學校的智慧財產權專業諮詢窗口 ，師生可以 call out 方式向律師詢問解答。每月並排定一次律師蒞校法律諮詢時段，提供師生面對面的完善諮詢服務。(學務處)

(5）訂定「元智大學網路使用管理辦法」，並確實（提示）告知使用者。

## 五網路管理

1．校內及學術網路使用、管理及安全維護皆透過「元智大學網路使用管理辦法」進行規範。本辦法明確定義校內及學術網路的使用規範，並載明違規事項，違反本辦法者，將視情節狀況交由相關主管單位議處。

## 元智大學培育智慧財產人員參與「能力認證」獎勵辦法

102.03.27 一○一學年度第七次研發處處務會議通過

## 第一條 目的

為縮短學用落差，培育本校智慧財產專業人員，增加本校師生產業競爭力，並落實研發處辦理培訓課程之成效，特訂定本辦法。

## 第二條 實施對象

參與研發處辦理之培訓課程並取得研習證書之本校教職員生。

## 第三條 補助項目

一、參與「TIPA 智慧財產培訓學院之智慧財產人員能力認證考試」，完成測驗者，予以各科報名費50％補助，同一科目僅補助1 次。

二、參與「TIPA 智慧財產培訓學院之智慧財產人員能力認證考試」，完成測驗並取得證書者，予以各科報名費100％補助，同一科目僅補助1 次。

## 第四條 申請方式

一、申請50%補助者：完成測驗後，應於取得成績單2 週內，填妥申請表並檢附參與研發處辦理之培訓課程之研習證書、報名費單據正本及成績單向研發處提出申請。

二、申請100%補助者：完成測驗後，應於取得證書2 週內，填妥申請表並檢附參與研發處辦理之培訓課程之研習證書、報名費單據正本、成績單及證書向研發處提出申請。

第五條 上述補助費用由研發處配合本校會計年度經費核銷期限規定，統一核發；研發處得視當學年度預算修正補助項目及獎勵內容。本辦法實施前已經檢定通過並取得合格證書者，不得追溯申請補助。

第六條 本辦法經研發處處務會議通過後施行。

## 元智大學網路使用管理辦法

91.01.21 90 學年度第 12 次行政會議通過

91.10.16 91 學年度第 4次行政會議修訂通過

97.06.30 96 學年度第 21次行政會議修訂通過

109.05.27 109 學年度第 22 次行政會議修訂通過

第一條 為規範本校網路資源使用、管理及安全維護，特訂定元智大學網路使用管理辦法((以下簡稱本辦法）。

第二條 本辦法適用於全校教職員生。

第三條 本校網路使用者所擁有之個人認證資訊，應妥善使用本校網路資源，並承擔因該帳號及密碼使用所衍生之一切法律責任。

## 第四條 使用規則

一、使用校園網路應遵守「臺灣學術網路管理規範」及「教育部校園網路使用規範」。

二、禁止盜用他人之個人認證資訊使用校園網路。

三、禁止利用網路傳送具威脅性、猥褻性、及不友善的資訊。

四、校園網路之使用主要目的為學術研究與教育行政之用途。

五、嚴禁冒（盜）用他人的 IP Address。

六、禁止使用高風險與或惡意之點對點(P2P)軟體與影響他人網路使用之檔案傳輸及阻礙。

七、禁止擅自截取網路傳輸訊息或窺視他人之電子郵件或檔案。

八、禁止隱藏帳號與隱匿 IP位址。

九、占用大量網路資源者，以致影響本校網路服務，資服處得視情況進行網路頻寬調節，以維持本校網路頻寬之傳輸順暢。

第五條 本校離職員工及不具本校學籍之學生，於離職生效或離校後，立即停用所有網路資源之使用。

第六條 基於協助防治網路違規及犯罪立場，相關單位請求對個人資料、檔案所為答覆、查詢、閱覽或複製本，均依相關規定辦理。

第七條 違反本辦法使用規則將立即停用所有網路之使用權。視情節由資服處檢具事實送至相關主管單位議處。

本辦法經行政會議通過後實施，修正時亦同。

# 元智大學伺服器管理辦法

101.04.30 100 學年度第 13 次行政會議通過

三、申請單位權責與義務

（一） 申請資格 :本校現職教師與職員。

四、各單位架設之伺服器，需使用合法授權軟體，並注意公開資訊內容的合法性。不得私自架設 VPN伺服器、IP 匿蹤伺服器，資服處得依情節輕重停止該伺服器申請與使用，若需要 VPN 服務，則統一使用資服處提供的 SSL VPN。

五、各單位於申請時，須檢附資訊安全防護計畫書備查，並每一年重新審核。

第四條 資服處定期對全校網際網路服務伺服器進行資安掃瞄檢測並產出報告文件，各單位伺服器管理人須依據資服處所提供之報告進行相關漏洞修補事宜。

第五條 本辦法經行政會議通過後實施，修正時亦同。

## 2.作業項目說明

2-1伺服器安全管理原則

(1)所有Windows 平台伺服器均需安裝伺服器防毒軟體，且病毒碼應適時更新至最新版本；Unix like平台視實際需求安裝防毒軟體。

(2)使用遠端管理軟體時須經身份驗證。

(3)由資服處外部進行伺服器維運管理，依據「資訊存取控制管理作業程序」辦理。

(4)由資服處內部進行伺服器維運管理，應考量以加密方式或限制特定來源連線使用。

(5)伺服器管理帳號與密碼須妥善保管，避免他人知悉。密碼的安全性及使用須符合至少 8碼，且須含數字、英文字母或特殊符號之規定。

(6)伺服器上目錄須依用途設定不同使用權限。

(7)嚴禁未經授權架設各類型伺服器如網站(Web)、檔案傳輸(FTP)、郵件(SMTP)等伺服器。

(8)禁止未經許可異動網路連線設定(如：新增網路卡、IP 等)。

(9)關閉非必要之通訊服務。

(10)系統需開啟系統日誌及適當稽核記錄功能，並將關鍵性系統伺服器之紀錄匯出存檔備查，並保留至少三個月。

(11)伺服器管理人員需確實檢視各伺服器運作狀態、系統日誌(含各式通訊服務之連線紀錄)、系統資源使用狀態、系統漏洞與修補程式安裝情形。

(12)系統設定檔應視情況予以備份處理。

(13)系統端應設定 session time out 時間，各伺服器與應用系統 session timeout 時間由承辦人自行設定。

(14)伺服器管理人員應定期監測硬體容量，容量使用率不得超過 85%，如達臨界值，應思考相對應變措施。

(15)系統安全性更新及修正

a.留意國內外危機處理中心(TWCERT)宣佈重大安全弱點及廠商發佈之修正檔訊息。

b.通知相關負責單位、人員取得修正程式。

c.與系統安全相關之修正程式應適時進行修補，必要時由廠商於測試後協助安裝；其他額外功能則視實際需求選擇性安裝。

(16)系統上線前應完成必要之漏洞修補作業，相關作業完成後，需關閉非必要網路服務。

(17)關鍵性伺服器於安裝修正程式前，需先行測試並詳細紀錄修正程序，確認系統穩定運作無誤後，方可於正式伺服器進行修正。

## 2-2資訊設備系統更新與漏洞修補

(1)資訊設備系統應定期進行系統弱點掃描並針對各設備發現之弱點執行必要性、可行性分析及修補之風險評估作業，以做為是否修補之依據。

(2)主機系統、網路設備、軟硬體廠商發佈之系統相關之修正程式（Patch），系統負責人應督導設備維護廠商執行修正程式之必要性、可行性分析及風險評估作業，進行修補。

(3)Windows server 則由系統管理人員評估必要性、可行性分析及風險評估作業後，安裝必要之 Service Pack，以避免系統安全漏洞發生。

## 2-3網路服務管理

(1)為維持網路持續正常運作，各重要網路通訊設備應有備援機制或備品及時替換或簽定維護廠商以維持網路服務。

(2)為確保網路服務持續暢通，應監測網路連線狀態。

(3)發生網路異常事件時，應經網路管理人員判斷處理後，必要時應通知相關單位協助處理。

(4)網路管理人員應於網路設備，限制未用之網路服務功能與通訊協定，僅開放其他正常的網路服務功能與通訊協定。

(5)外部廠商若應維護/維修需要，連線資服處內部網路，則須填寫「廠商連線申請單」，並由相關管理人進行評估，確定可行且無安全上的顧慮後，經由權責主管同意方得開放。

(6)發生網路安全事件，依據「資訊安全事件管理暨通報作業程序」辦理。

(7)對任何網路安全事件，網路管理人員應立即向權責主管反應。

## 2-4網路區隔管理

(1)網路之使用，應對使用狀況與條件，區隔不同網路區段，以便進行網路存取控管。

(2)為達成網路使用控制及管理考量，應鑑別主機與使用者工作內容，賦予適當網路區段之 IP 位址。

## 2-5IP 位址分配原則

(1)本校單位 IP 位址區段的規劃，概由本處統一管理，網路使用者應依所分配之IP 位址區段進行使用，不得任意佔用其他使用者之 IP 位址區段，以維護校園網路內各使用者之合法權益。

(2)本校單位現有 IP 不足，需新增 IP 區段時，由該單位提出申請填寫「網段申請單」，並由資服處網路管理人員進行評估，依據使用需求及校內 IP位址可用數量適當分配。

(3)網路管理人員應對 IP 位址區段進行管制，並建立 IP 位址區段暨各單位權責管理人員連絡清單，以利日後追蹤管理。

(4)資服處委託代為管理 IP 位址之單位管理人員，取得本處分配的 IP 位址區段後，單位內 IP 位址的發放及分配方式由該單位自行訂定。

## 2-6網路流量管理

(1)應記錄網路流量使用現況，瞭解網路頻寬使用情形。

(2)如該區段有流量或連線異常之情況，應立即查明原因，並採取適當措施。

## 2-7網路使用者管理

(1)網路使用者應遵守「元智大學網路使用管理辦法」規定，並確實瞭解所須負的責任，如有違反網路安全情事，依相關法規處理。

(2)被授權的網路使用者，只能在授權範圍內使用網路。

(3)不得將自己的登入身份識別與密碼交付他人使用。

(4)禁止以任何方法竊取他人的登入身份識別與密碼。

(5)禁止以任何儀器設備或軟體工具竊聽網路上的通訊。

(6)不得以任何手段蓄意干擾或妨害網路的正常運作。

(7)需開啟防毒軟體之網頁防護功能。

## 2-8 網路安全監控

(1)路由交換器設備、資安設備應啟用系統日誌紀錄(syslog)，同步轉錄至log server，系統日誌紀錄保留至少三個月。

(2)監控及檢核過程中若發現重大安全事件「資訊安全事件管理暨通報作業程序」通報程序辦理。

## 2-9網路入侵之偵測與處理

(1)網路如發現有被入侵或有疑似被侵入情形，依據「資訊安全事件管理暨通報作業程序」通報程序辦理。

(2)需對網路入侵時之相關紀錄進行蒐證，以便進行後續追查作業。

(3)對入侵者的追查，除利用稽核檔案提供的資料外，可用系統指令執行反向查詢。

(4)應全面檢討網路安全措施或修正防火牆的設定，以防禦類似的入侵與攻擊。

(5)立即向權責主管人員報告入侵情形。

## 2-10路由交換器/防火牆建置與管理

(1)路由交換器設備與防火牆之設定檔，應每季備份。

(2)內部與委外人員進行維護作業均應建立紀錄。

(3)網管人員應定期檢視路由交換器與防火牆設備是否存在漏洞，若有漏洞，應立即通知廠商或取得合法修正程式後，進行修補。

(4)網路設備需關閉不必要服務(Service)，預防不明人士透過網路偵測工具，對其漏洞展開攻擊，造成網路設備無法正常運作。

(5)路由交換器設備與防火牆進行設備配置與異動時應予以紀錄。並於每次修改設定檔後進行備份，並需有適當的回復程序。

(6)防火牆管理之權責主管應管制防火牆管理人員帳號數量，僅授予合法管理者及代理人。

(7)路由交換器與防火牆設備管理帳號與密碼須妥善保管，避免他人知悉。密碼的安全性及使用須符合至少 8碼，且須含數字、英文字母或特殊符號之規定。

## (8)路由交換器遠端存取原則

a.應於各路由器中設定適當之 ACL，限制遠端存取路由器之來源 IP 位置。

b.為維護目的，需使用遠端存取。網路設備需啟用 SSH 以替代未加密之TELNET 管理路由器。若無法啟用 SSH，應於安全之連線環境下，使用 TELNET 連線。

c.委外廠商若有遠端維護之需求，須於各路由器中設定適當之 ACL限制遠端路由器之來源 IP 位置。

d.為避免路由器遭受入侵與滲透，除因特殊需求，否則應限制來自網際網路之遠端登入管理方式。

## (9)路由交換器 Access-List 設定

a.依照連接對象及網路協議相關事宜訂定 Access List 加以過濾。

b.依據路由器負載程度，將已知可能造成網路系統無法運作之通訊埠訂定Access List 加以過濾。

c.透過存取控制過濾功能，限制未經授權之連線，自遠端連接路由設備。

(10)路由交換器 SNMP 設定啟用 SNMP 協定前，需將 community string 之 Read-Only 與 Read-Write修改與原廠預設值不同的 community string。

(11)防火牆管理原則a.防火牆管理人員須在指定 IP 位址，並透過特定的連接埠登入管理。b.針對重要網路服務主機群應以防火牆連接校園網路，限制存取服務。若防火牆效能允許，應保留 3個月連線紀錄。c.位於內部網路重要網路服務主機群之防火牆，應以防火牆規則正向表列，除開放對外服務通訊埠或 URL外，其餘應予關閉。

(12)防火牆安全規則設定與異動管理

a.應避免不當範圍之群組設定，包含連線來源、目標位址與通訊協定。

b.防火牆規則若為正向表列，應於規則列表最後設定拒絕全部服務的規則(Implicit Denied All)。

c.防止防火牆因負載過重而效能不佳，應避免邏輯重複之安全規則。

d.網路防火牆需求異動。資服處主機伺服器管理人員應填寫「網路防火牆管制申請單」提出異動需求，防火牆管理人員須於權責主管核可後方可執行相關異動作業。

e.網頁應用程式防火牆 Profile之 Policy 異動。由本處系統管理人員填寫「網頁防火牆管制申請單」提出異動需求，須於權責主管核可後，由系統管理人員執行相關異動作業，並將「網頁防火牆管制申請單」交由防火牆管理人員保管。

f.網頁應用程式防火牆設定異動：由資服處系統管理人員填寫「網頁防火牆管制申請單」提出異動需求，防火牆管理人員須於權責主管核可後方可執行相關異動作業。

g.防火牆管理人員建立或更改安全規則時，應對規則加註，並保留設備之備份檔為最新狀態。

h.異動規則前應測試推演以確認可行性，並考量是否可與其他規則整併。

i.為求落實安全管理，於防火牆安全規則建置後，應每半年由防火牆管理人員、權責主管會同主機、應用系統相關人員審查規則適用性，並予以檢討修正，避免因時間或業務變更而不符現狀。

## 2-11紀錄與蒐證

依照「資訊安全事件管理暨通報作業程序」辦理。

## 柒、控制重點：

1.系統是否開啟系統日誌及適當稽核記錄功能。

2.關鍵性系統伺服器之紀錄是否匯出存檔備查，保留至少三個月。

3.伺服主機是否有確實更新 Service Pack 及病毒碼至最新。

4.伺服主機是否有確實檢核非合法授權安裝狀況。

5.伺服主機管理人員是否確實進行系統運作等狀況檢核，並記錄之。

6.各系統提供服務時是否有設定 session time out 時間，以避免使用者長時間未登。

7.上線伺服主機進行漏洞更新時，是否有事先對更新套件測試，確保無誤後才予以更新。

8.發生網路安全事件是否確實依規定處理，及向權責主管反應。

9.外部人員進行資訊系統、網路維護時，是否有確實填寫廠商連線申請單。

10.是否有記錄網路流量使用現況，以瞭解網路頻寬使用情形。

11.路由交換器等重要設備是否有啟動 log 機制，並保留三個月記錄。

12.路由交換器等重要設備是否有定期備份相關設定檔。

13.路由交換器等重要設備是否有限制人員管理權限。

14.在連線至路由交換器等重要設備時，是否僅限定 SSH協定或有控管之連線。

15.路由交換器等重要設備之 SNMPCommunity String 值是否有確實修改，而非預設值。

16.防火牆之管理連線是否有做安全性之控管。

17.防火牆之正向表列是否依實際狀況訂定，無擴大使用之疑慮。

18.防火牆申請是否有確實申請並經權責主管同意。

19.防火牆規則是否定期檢視其適用性。
